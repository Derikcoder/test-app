PERSONA SUMMARY
- Persona ID: CUST-003
- Persona Name: Group Business Single Ownership (Multi-Branch Company)
- Persona Type: Customer
- Classification: Company with multiple branches/locations, single owner/parent entity
- Objective: Coordinate and approve services across branches, maintain centralized control over payments and vendor relationships
- Profile Model: Decentralized service booking + centralized approval & payment
- Login credentials: use entities/personas/LOCAL_TEST_CREDENTIALS.md (local-only, gitignored).

ORGANIZATIONAL STRUCTURE
- Parent Company / Headquarters: Central billing, approval authority, payment control
- Branch Offices: Can request services independently, require HQ approval before proceeding
- Service Workflow: Branch → Approval Request → HQ Decision → Order Number Assignment → Service Execution
- Payment Authority: Headquarters only (all invoices paid from central account)
- Reporting: Consolidated by HQ, detailed by branch

PROFILE PAGE FEATURES & UAT REQUIREMENTS

=== 1. BRANCH SELECTOR & MULTI-SITE VIEW ===
Profile section: "My Organization" / Branch Navigation
- Top-level control: Branch dropdown or tab switcher
- Available Branches:
  * Headquarters / Main Office (central control functions)
  * Branch A, Branch B, Branch C, ... (service request branches)
- Navigation:
  * Selecting a branch shows branch-specific views (equipment, service history, quotes)
  * Selecting "Headquarters" shows consolidated/company-wide views
- Validation Points:
  * Each branch has its own data context
  * No data leakage between branches unless explicitly authorized
  * Branch selector is persistent across profile navigation

=== 2. BRANCH-LEVEL: EQUIPMENT & MAINTENANCE SCHEDULE ===
Profile section: "Equipment & Maintenance" (Branch View)
- Same as single-business, but scoped to selected branch
- For each branch:
  * Equipment list is unique to that branch site
  * Service history is specific to that branch's assets
  * Maintenance schedules are tracked per branch
- Validation Points:
  * Equipment list updates correctly when switching branches
  * No cross-branch equipment overlap
  * Branch maintenance alerts are independent

=== 3. BRANCH-LEVEL: SERVICE HISTORY & PENDING APPROVALS ===
Profile section: "Service History & Approvals" (Branch View)
- Two subsections:
  a) Pending HQ Approval: Service requests awaiting headquarters decision
     - Shows: Request date, equipment/category, cost estimate, request status, order number (blank - awaiting HQ assignment)
     - Action buttons: "View Request Details", "Add Notes"
     - Branch cannot proceed until HQ approves
  b) Service History: Completed services and approved requests
     - Shows: Service date, equipment, agent, cost, order number (assigned by HQ), status
     - Organized by month or asset
- Approval Request Detail View:
  * Equipment requiring service and fault description
  * Estimated cost (from agent quote)
  * Branch contact who submitted request
  * Branch operations manager approver
  * Notes field (branch can add context for HQ)
  * HQ approval status: "Pending", "Approved", "Rejected", "More Info Requested"
  * Timestamp of request and HQ response
- Validation Points:
  * Requests are assigned to correct branch
  * Estimates are accurate
  * Approval status is consistently tracked
  * Order number is populated only after HQ approval
  * Branch cannot modify estimates or proceed without approval flag

=== 4. HEADQUARTERS: SERVICE APPROVAL QUEUE ===
Profile section: "Approval Queue" (HQ/Headquarters View)
- Centralized dashboard for all pending service requests across all branches
- Approval Queue Table:
  * Branch name
  * Equipment type and issue
  * Requested cost estimate
  * Request date and time elapsed
  * Branch contact name
  * Action buttons: "Approve & Assign Order #", "Request More Info", "Reject"
- Approval Workflow:
  1. HQ manager reviews request
  2. HQ enters order number (from customer's internal PO system, cost center, or budget code)
  3. HQ clicks "Approve & Assign Order #"
  4. System confirms approval and notifies branch
  5. Service call can now be initiated with order number included
- Validation Points:
  * Only HQ users can approve or reject
  * Order number is required to approve (not optional)
  * Order number must be unique within customer system (validation check if available)
  * Approval notifications reach correct branch contact
  * Rejected requests include reason for rejection

=== 5. HEADQUARTERS: CONSOLIDATED FINANCIAL STATEMENT ===
Profile section: "Company Financial Statement" (HQ View)
- Consolidated view across all branches:
  * Total company spend (YTD)
  * Spend by branch (showing each branch's total expenditure)
  * Spend by category across all branches
  * Spend by service provider across company
  * Total pending approvals (sum of all pending requests across branches)
  * Payment compliance (company-wide, on-time payment %)
  * VAT tracking (total VAT paid across all branches)
  * Cost trend (YTD vs. prior year for company)
  * Budget utilization by branch (if budget data available)
- Detailed Breakdown:
  * Branch A: R50,000 YTD (Mechanical: R30,000, Electrical: R15,000, Plumbing: R5,000)
  * Branch B: R35,000 YTD (Electrical: R25,000, HVAC: R10,000)
  * Branch C: R22,000 YTD (Plumbing: R22,000)
  * Company Total: R107,000 YTD
- Validation Points:
  * Totals are mathematically correct (sum of branches = company total)
  * Each invoice is counted only once
  * VAT is correctly summed
  * Payment compliance is accurate
  * Budget tracking (if enabled) reflects actual vs. planned spend

=== 6. HEADQUARTERS: CONSOLIDATED SERVICE PROVIDER PERFORMANCE ===
Profile section: "Company Service Providers & Performance" (HQ View)
- Consolidated ranking of all agents across all branches
- For each provider:
  * Agent name and specialty
  * Total services completed across all branches
  * Services per branch (breakdown, e.g., "Branch A: 5 services, Branch B: 3 services")
  * Overall rating (average from all branch reviews)
  * Response time average (across all branches)
  * Preferred vendor status (company-wide)
  * Last service date (most recent across all branches)
  * Branch specialists (if agent only services specific branches)
- Validation Points:
  * Frequency counts are accurate across all branches
  * Ratings are properly weighted or averaged
  * Preferred vendor status is company-wide (not per-branch)
  * Response times reflect company-wide performance

=== 7. BRANCH-LEVEL & HQ: REVIEWS & VENDOR MANAGEMENT ===
Profile section: "Service Provider Reviews" (Branch and HQ Views)
- Branch Level:
  * Branch can review service providers who worked on branch services
  * Branch review is submitted as feedback
  * Reviews are visible to HQ but may not affect company-wide rating
- HQ Level:
  * HQ can promote providers to "preferred vendor" status
  * HQ can view all branch reviews
  * HQ can aggregate ratings for company-wide performance
- Review Workflow (unchanged):
  * Service completion → Review opportunity
  * Star rating, category ratings, text feedback
  * Recommend for future services? (Y/N)
- Validation Points:
  * Branch reviews do not lock HQ from making vendor decisions
  * HQ ratings can be independent of or informed by branch ratings
  * Preferred vendor status is enforced company-wide

=== 8. INVOICE STRUCTURE FOR MULTI-BRANCH COMPANY ===
Key Addition: Order Number (Customer-Supplied)
- Invoice must include:
  * `order_number` (string, supplied by customer, not generated by service provider)
  * `order_source` (optional: "Customer ERP", "Manual Entry", "API", etc.)
  * `approval_reference` (reference to HQ approval record)
  * `branch_identifier` (which branch requested the service)
- Order Number Rules:
  * Cannot be null (must be assigned by HQ before service)
  * Must be unique within customer's system (validation depends on customer's rules)
  * Typically format: cost center code, PO number, budget code, or internal ref
  * Is used for cost allocation, audit trail, and purchase order matching
- Invoice Example (v1 structure):
  ```
  {
    "order_number": "CC-2026-05-BRANCH-A-0047",
    "order_source": "Customer ERP System",
    "approval_reference": "APPR-001109",
    "branch_identifier": "BRANCH-A",
    "branch_site_address": "123 Main St, Johannesburg",
    ... rest of invoice fields ...
  }
  ```
- Validation Points:
  * Order number is present on all invoices for this customer type
  * Order number matches HQ approval record
  * Branch identifier matches service location
  * Order number is immutable once invoice is issued (no editing)
